## Summary

Fixed four runtime correctness issues from a senior-engineer code review: frame-rate-dependent meteor motion in `Neon Meteor Run`, stale sync freshness labeling on the home page, missing later-wave banner feedback in `Pixel Breach`, and a hardcoded featured status badge in the hero card.

## Why

These bugs affected shipped behavior rather than code style. Two of them made UI state inaccurate, one made gameplay depend on machine frame rate, and one hid correct manifest status data from the featured cabinet surface.

## How

Converted the `Neon Meteor Run` meteor update path to use `dt`, added a minute timer on `HomePage` so the sync pill refreshes between feed polls, introduced wave-intro state in `Pixel Breach` for each post-wave transition, bound the hero status badge to manifest data, cleaned the remaining corrupted sync text, and added focused regression tests for sync freshness and hero status rendering.

## Changes Made

- Changed `NeonMeteorRunGame` meteor position and rotation updates to multiply by `dt`.
- Added minute-level timer state to `HomePage` so the sync pill can advance from `1m ago` to `2m ago` without waiting for a feed refetch.
- Added `waveIntroTime` state in `PixelBreachGame` and reset it when a new wave starts so later wave banners render again.
- Replaced the hero card's hardcoded `Ready` badge with a manifest-driven status badge.
- Rebuilt `LiveSyncPill.tsx` with clean text output to remove remaining mojibake in the sync UI.
- Added `Hero.test.tsx` and `HomePage.test.tsx` regression coverage.
- Bumped the project version to `0.4.2` and updated docs/changelog/worklog links.

## Problems Encountered

- The new sync-label regression test initially failed because React timer-driven updates were not wrapped in `act(...)`.
- `HomePage.tsx` triggered a fast-refresh lint warning when the sync-label helper was exported for testing.
- The sync pill file still contained corrupted punctuation, which also made the initial text assertion brittle.

## Solutions

- Wrapped timer advancement in `act(...)` in the test.
- Kept the sync-label helper file-local instead of exporting it.
- Replaced the sync pill file content with a clean ASCII-safe version and updated assertions to match rendered text robustly.

## Bugs Found

- `Neon Meteor Run` meteor movement and spin were frame-rate dependent.
- Home-page sync freshness text stopped updating after render.
- `Pixel Breach` later waves never re-triggered the wave banner.
- The hero card could misreport a featured cabinet as `Ready` even when the manifest said otherwise.
- The sync pill still had visible corrupted text.

## Bug Fixes

- Applied time-based meteor updates.
- Added scheduled re-rendering for freshness text.
- Added per-wave intro timing for `Pixel Breach`.
- Bound the featured badge to `featuredGame.status`.
